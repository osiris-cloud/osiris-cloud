LOGGING IN TO YOUR REGISTRY

To log in to the registry, first you need to create an access key. It is recommended that you limit the scope of this key to Container Registry and subscope to Registry Login Only for security. After you copied the token, you can log in to the registry using the following command. Paste the token when prompted for the password.

docker login -u osiris {{ cr.domain }}

BUILDING YOUR IMAGE

Building packs your application and its dependencies into a reusable template called an image. Replace <my-image> with the name of your image. Optionally, you can specify a tag. If you don't specify one, Docker will use latest.

Assuming you have a Dockerfile in the current directory, you can build the image using the following command.

docker build -t <my-image> .

PUSHING YOUR IMAGE

To push an image to the registry, first tag the local image to point to osiris cloud. Replace <my-image> with the name of your image. If you push a new image using an existing tag, the old image will be replaced.

docker tag <my-image> {{ cr.url }}/<my-image>

After you tag, use the push command to upload the image. Replace <my-image> with the name of your image.

docker push {{ cr.url }}/<my-image>
